## ⚪ 生徒の得点記入・証拠ファイルアップロード機能

**概要**
生徒が LIFF 画面から得点・順位を手動入力するだけでなく、証拠写真や成績関連ファイルをアップロードできるようにする。

**機能一覧**

| 機能 | 内容 |
|------|------|
| 得点・順位の手動入力 | 既存機能の改善・UI 整備 |
| 証拠写真アップロード | 得点・順位の裏付け写真（試験返却物など） |
| 評定アップロード | 通知表・成績証明書などの画像・PDF |
| その他ファイルアップロード | 任意の写真・PDF（用途フリー） |

**設計イメージ**
- アップロード先: Google Drive の校舎ごとのフォルダ（子SS の config シートに `DRIVE_FOLDER_ID` を保持）
- GAS 側: `DriveApp.getFolderById().createFile(blob)` でファイル保存
- ファイル情報（Drive ファイルID・種別・生徒ID・試験ID）を子SS の `student_files` シートに記録
- フロント（LIFF）: `FileReader` で base64 → GAS に送信（画像・PDF 対応）
- 閲覧: 生徒は自分のファイルのみ、管理者は校舎全体を閲覧可能

**新規シート（子SS）**
```
student_files
  file_id       - 自動生成
  student_id    - 生徒ID
  exam_id       - 試験ID（紐付ける場合）
  file_type     - score_proof / evaluation / other
  drive_file_id - Google Drive のファイルID
  uploaded_at   - アップロード日時
```

**考慮事項**
- GAS の `google.script.run` は送受信データ上限 50MB
- 画像は送信前にクライアント側で Canvas API を使って圧縮する
- PDF は分割が必要になる場合あり
- Drive フォルダの共有設定（生徒は write のみ・read 不可、または制限付き）
- 管理者画面からもファイル一覧・プレビューができると望ましい

## ⚪ 教科ごとの満点設定・バリデーション強化

**概要**
現状の得点バリデーション（0〜200）は全教科共通の暫定実装。
将来的に学校・教科ごとに満点を設定し、その値でバリデーションする。

**設計イメージ**
- 満点の管理: `pattern_subjects` または専用シートに `max_score`（学校・教科単位で設定）
- バリデーション: フロント（即時フィードバック）とサーバー（`saveAllScores`）の両方で実施
- フロントに `max_score` を渡すために `getData.js` の戻り値を拡張

## ⚪ 生徒入力による表記ゆれ対応（パターン統合機能）

**概要**
生徒が初期設定フローでコースを手入力するため、「普通」「普通科」「普通（科）」のような表記ゆれが発生する。
これにより、同一学校・同一コースのつもりが別パターンとして分裂してしまうケースに対応する。

**機能イメージ**
- 管理画面で既存の school_course / school_name のパターン一覧を確認し、統合先を指定して「マージ」できる
- 統合時: `exam_patterns`・`pattern_subjects`・`students_master` 内のコース名を一括置換
- 重複した `exam_patterns` は削除（または統合先にリダイレクト）
- `school_course_master` の古いエントリも削除

**考慮事項**
- 統合は不可逆操作なので確認ダイアログ必須
- `scores_data` は `exam_id` 参照なので `exam_patterns` の再構成が必要な場合は慎重に
- 将来的には入力補完（既存コース名サジェスト）でそもそも表記ゆれを防ぐ方が根本解決
